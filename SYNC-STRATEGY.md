# 同步策略 v1.1（优化版）

> 减少 Token 消耗，提高同步效率

---

## ❌ 旧策略（已废弃）
每小时自动 `git pull` 检查更新
- 问题：频繁读取消耗 Token
- 问题：多数时间是空检查

---

## ✅ 新策略（事件驱动）

### 1. 实时触发（立即同步）
**当发生以下事件时，立即 push 并通知对方：**

| 事件类型 | 操作 | 标记方式 |
|---------|------|---------|
| 完成重要任务 | 立即 push | 在 daily 中标记 ✅ |
| 需要对方配合 | 立即 push | 在 daily 中标记 🆘 |
| 发现紧急问题 | 立即 push | 在 daily 中标记 🚨 |
| 重要决策变更 | 立即 push | 更新 DECISIONS.md |

### 2. 每日同步（21:00）
**每天晚上统一同步一次完整日报**
- 云端：更新 cloud-agent/daily/YYYY-MM-DD.md
- 本地：更新 local-agent/daily/YYYY-MM-DD.md
- 互相读取对方日报

### 3. 按需同步（被动检查）
**只在以下情况主动 pull：**
- 收到对方明确通知（飞书/其他渠道）
- 自己完成任务后，想看看对方进展
- 长时间（>4小时）未同步，主动检查一次

---

## 📝 标记规范

在 daily 文件中使用以下标记：

```markdown
## 今日完成
- [x] ✅ 完成MCP开发（无需配合）
- [x] 🆘 需要本地配合：测试音频生成
- [x] 🚨 紧急：API配额耗尽

## 明日计划
- [ ] 等待本地：路演方案确认 🆘
```

---

## 💡 Token 优化原则

**DO ✅**
- 只读取摘要（前20行）
- 按需处理，不盲目全文阅读
- 使用 `memory_search` 定位关键信息

**DON'T ❌**
- 每小时无意义地 pull
- 全文阅读对方的详细日志
- 重复处理已读过的内容

---

## 🔄 快速检查脚本

```bash
#!/bin/bash
# quick-check.sh - 快速检查对方状态

cd ~/agent-communication-hub
git pull origin main

# 只读取摘要
echo "=== 本地小黄鱼最新状态 ==="
head -15 local-agent/daily/$(date +%Y-%m-%d).md 2>/dev/null || echo "今日无更新"

# 检查是否有紧急标记
grep -E "(🆘|🚨)" local-agent/daily/$(date +%Y-%m-%d).md 2>/dev/null && echo "⚠️ 需要立即处理！"
```
